## Git und GitHub
 
### Projekt klonen
Das GitHub-Projekt wurde mit folgenden Befehlen auf Kali geklont:
```bash
    git clone https://github.com/aldin-122/4AHITS-ITSE-MUKIC.git
```

Danach ging ich hinein um weiter fortzufahren:
```bash
    cd 4AHITS-ITSE-MUKIC
 ```

### Änderung in VS Code
Das Projekt wurde mit VS Code geöffnet:
```bash
    code .
 ```

Eine Markdown-Datei (z. B. 250908.md) wurde bearbeitet und eine Teständerung eingefügt.  
Anschließend im Terminal:
```bash
    git add .
    git config --global user.email "aldin.mukic@htl-braunau.at"
    git config --global user.name "Aldin Mukic"
    git commit -m "Teständerung im Labor"
    git push
 ```

Git verlangte die Konfiguration von Benutzername und E-Mail, bevor der erste Commit möglich war.  
Nach der Konfiguration funktionierte der Push erfolgreich.

## Kontrolle auf GitHub Pages

Damit wurde bestätigt, dass der Push und die Seitenaktualisierung korrekt funktionieren.

## Änderung über die GitHub-Oberfläche
Eine zweite Änderung wurde direkt auf GitHub vorgenommen (Markdown-Datei bearbeitet und committed).  
Diese Änderung wurde anschließend in Kali übernommen mit:
```bash
    git pull
 ```

Der Inhalt der Datei wurde erfolgreich aktualisiert – die Synchronisation zwischen lokalem Klon und Remote-Repository funktioniert somit in beide Richtungen.
