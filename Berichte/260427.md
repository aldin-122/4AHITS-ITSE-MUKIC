# Linux Permissions Übungen

## Vorbereitung der Übungsumgebung

### Aufgabenstellung
Das Skript soll die Laborumgebung einrichten. Es legt Gruppen und User an,
setzt Passwörter und erstellt die nötige Ordnerstruktur.

Das Skript macht folgendes:
- Erstellt 3 Gruppen: music, wrestle, emperors
- Erstellt 4 User (elvis, blondie, hogan, nero) und weist sie den Gruppen zu
- Setzt alle Passwörter auf 'kali'
- Erstellt den Ordner /tmp/lab/ mit Unterordnern und ein paar Testdateien
- Setzt Besitzer und Rechte auf root mit 755

### Lösung
Das Skript wurde als User kali mit sudo ausgeführt:


Nach der Passwort-Eingabe für kali wurde das Skript ohne Fehler ausgeführt
und die Laborumgebung wurde erfolgreich eingerichtet.

## 1 Übung (Analyse und Symbolic Mode)

### 1. Rechte auslesen

#### Aufgabenstellung
Ermittle die Permissions, den User Owner und den Group Owner der Datei
/tmp/lab/wrestle_plans.txt.

#### Lösung

```bash
ls -l /tmp/lab/wrestle_plans.txt
```

Ausgabe:
```bash
-rwxr-xr-x 1 root root 6 Apr 27 05:55 /tmp/lab/wrestle_plans.txt
```

- Permissions: -rwxr-xr-x
- User Owner: root
- Group Owner: root
- user (u): lesen, schreiben, ausführen
- group (g): lesen, ausführen
- others (o): lesen, ausführen

---

### 2. Private Files

#### Aufgabenstellung
Erstelle eine Datei notes.txt und entziehe der other-Klasse das Read-Recht
mit dem symbolischen Modus von chmod.

#### Lösung

```bash
touch notes.txt
chmod o-r notes.txt
ls -l notes.txt
```

Ausgabe:
```bash
-rw-rw----
```

- user (u): lesen, schreiben
- group (g): lesen, schreiben
- others (o): keine Rechte

---

### 3. Group Sharing

#### Aufgabenstellung
Setze die Permissions für notes.txt so, dass group lesen und schreiben darf,
user alle Rechte behält und others keinen Zugriff hat. Alles in einem
chmod Aufruf.

#### Lösung

```bash
chmod u=rwx,g=rw,o= notes.txt
ls -l notes.txt
```

Ausgabe:
```bash
-rwxrw----
```

- user (u): lesen, schreiben, ausführen
- group (g): lesen, schreiben
- others (o): keine Rechte

---

### 4. Skript-Vorbereitung

#### Aufgabenstellung
Gib user und group das Execute-Recht für die Datei myscript.sh.

#### Lösung

```bash
touch myscript.sh
chmod ug+x myscript.sh
ls -l myscript.sh
```

Ausgabe:
```bash
-rwxrwxr--
```

- user (u): lesen, schreiben, ausführen
- group (g): lesen, schreiben, ausführen
- others (o): lesen

### 5. Kollektive Änderung

## 2 Übung (Ownership und Gruppenkollaboration)

### 1. Group Change

#### Aufgabenstellung
Wechsle zum User blondie und erstelle die Datei /tmp/lab/song.txt.
Ändere den Group Owner auf music.

#### Lösung

```bash
su - blondie
touch /tmp/lab/song.txt
chgrp music /tmp/lab/song.txt
ls -l /tmp/lab/song.txt
```

Ausgabe:
```bash
-rw-rw-r-- 1 blondie music /tmp/lab/song.txt
```

- User Owner: blondie
- Group Owner: music

Hinweis: Zuerst musste mit `sudo chmod 777 /tmp/lab` der Ordner für alle
schreibbar gemacht werden, da sonst blondie keine Dateien erstellen konnte.

---

### 2. Unterschied su vs su -

#### Aufgabenstellung
Experimentiere mit dem Unterschied zwischen su und su -.

#### Lösung

```bash
su hogan        # nur User wechseln
pwd             # zeigt /home/blondie
echo $HOME      # zeigt /home/hogan

exit

su - hogan      # kompletter Login
pwd             # zeigt /home/hogan
echo $HOME      # zeigt /home/hogan
```

Bei `su` bleibt man im Verzeichnis des vorherigen Users, die Umgebung
wird nicht komplett neu geladen. Bei `su -` landet man direkt im
Home-Verzeichnis des neuen Users mit seiner eigenen Umgebung.

---

### 3. User Transfer

#### Aufgabenstellung
Mache elvis zum User Owner der Datei /tmp/lab/song.txt.

#### Lösung

```bash
sudo su
chown elvis /tmp/lab/song.txt
ls -l /tmp/lab/song.txt
```

Ausgabe:
```bash
-rw-rw-r-- 1 elvis music /tmp/lab/song.txt
```

- User Owner: elvis
- Group Owner: music

`chown` braucht root-Rechte, da nur root den User Owner einer Datei
ändern kann.

## 3 Übung (Directory Permissions und Octal Notation)

### 1. Oktale Umwandlung

#### Aufgabenstellung
Gib die chmod-Befehle mit Oktalwerten für folgende Zielzustände an:
- rwxr-xr-x
- rw-rw----
- rwx------

#### Lösung

r=4, w=2, x=1 → addieren pro Klasse:

- `rwxr-xr-x` -> chmod 755
- `rw-rw----` -> chmod 660
- `rwx------` -> chmod 700

---

### 2. Pub-Verzeichnis

#### Aufgabenstellung
Erstelle /home/kali/pub und setze die Rechte so: Besitzer darf alles (7),
Gruppe darf auflisten und wechseln (5), others darf nur wechseln (1).

#### Lösung

```bash
mkdir /home/kali/pub
chmod 751 /home/kali/pub
ls -ld /home/kali/pub
```

Ausgabe:
```bash
drwxr-x--x 2 kali kali
```

- user (u): lesen, schreiben, ausführen → 7
- group (g): lesen, ausführen → 5
- others (o): nur ausführen (wechseln) → 1

---

### 3. Recursive Lockdown

#### Aufgabenstellung
Mache /tmp/lab/drafts/ und alle darin enthaltenen Dateien rekursiv für
group lesbar und schreibbar, others soll keinen Zugriff haben.

#### Lösung

```bash
chmod -R g=rw,o= /tmp/lab/drafts/
ls -l /tmp/lab/drafts/
```

Ausgabe:
```bash
-rwxrw----
```

Mit `-R` werden die Rechte rekursiv auf alle Dateien und Unterordner
angewendet.

---

### 4. Search Permission

#### Aufgabenstellung
Setze die Rechte auf /home/kali so, dass andere User keine Dateien
auflisten können, aber in /home/kali/pub wechseln dürfen.

#### Lösung

```bash
chmod 701 /home/kali
ls -ld /home/kali
```

Ausgabe:
```bash
drwx------x 22 kali kali
```

- user (u): lesen, schreiben, ausführen → 7
- group (g): keine Rechte → 0
- others (o): nur ausführen (wechseln) → 1

others kann nicht mit `ls` den Inhalt auflisten, aber mit `cd` in
/home/kali/pub wechseln.

---

### 5. Analyse

#### Aufgabenstellung
Warum können User nach chmod -R 660 /tmp/lab/ nicht mehr mit cd in
Unterverzeichnisse wechseln?

#### Lösung

660 entspricht `rw-rw----`, das Execute-Recht (x) fehlt komplett.
Bei Verzeichnissen bedeutet x nicht ausführen sondern betreten.
Ohne x kann man mit cd nicht mehr in Unterverzeichnisse wechseln.

## 4 Übung (Webserver Directory Permissions)


### Aufgabenstellung
Experimentiere mit den Permissions des /var/www Verzeichnisses und
beobachte die Auswirkungen von r und x bei Verzeichnissen.

### Lösung

#### Ausgangszustand
```bash
ls -ld /var/www
```
Ausgabe:
```bash
drwxr-xr-x 3
```

---

#### other/r entfernen
```bash
sudo chmod o-r /var/www
ls -ld /var/www
```
Ausgabe:
```bash
drwxr-x--x
```

**Kann kali den Inhalt auflisten?**
```bash
ls /var/www
```
Ausgabe:
```bash
ls: cannot open directory '/var/www': Permission denied
```
Nein, da `r` fehlt kann kali den Inhalt nicht auflisten.

**Ist cd /var/www noch möglich?**
```bash
cd /var/www
pwd
```
Ausgabe:
```bash
/var/www
```
Ja, da `x` noch gesetzt ist kann kali das Verzeichnis betreten.

**Kann in html gewechselt werden?**
```bash
cd /var/www/html
pwd
```
Ausgabe:
```bash
/var/www/html
```
Ja, in Unterverzeichnisse wechseln ist noch möglich da `x` gesetzt ist.

---

#### other/r zurücksetzen, other/x entfernen
```bash
sudo chmod o+r /var/www
sudo chmod o-x /var/www
ls -ld /var/www
```
Ausgabe:
```bash
drwxr-xr--
```

```bash
ls /var/www
cd /var/www
```
Ausgabe:
```bash
ls: cannot access '/var/www/html': Permission denied
html
cd: permission denied: /var/www
```

- `ls` zeigt zwar dass `html` existiert, aber kann keine Details
  abrufen da `x` fehlt
- `cd` ist nicht mehr möglich da `x` fehlt, betreten ist nicht erlaubt

---

#### Rechte wieder herstellen
```bash
sudo chmod o+x /var/www
ls -ld /var/www
```
Ausgabe:
```bash
drwxr-xr-x
```

**Zusammenfassung:**
- `r` bei Verzeichnissen → Inhalt auflisten mit `ls`
- `x` bei Verzeichnissen → Verzeichnis betreten mit `cd`
- Ohne `x` ist kein `cd` möglich, auch wenn `r` gesetzt ist
- Ohne `r` ist kein `ls` möglich, aber `cd` funktioniert noch

## 5 Übung (Webserver Directory Permissions II)

### Aufgabenstellung
Erstelle /var/www/data und sorge dafür dass nur User der Gruppe www-data
das Verzeichnis verwenden können. Others soll den Inhalt auflisten aber
keine Dateiinhalte lesen können. User der Gruppe www-data können
Dateiinhalte anderer User ansehen aber nicht verändern.

### Lösung

#### Verzeichnis erstellen und Gruppe setzen
```bash
sudo mkdir /var/www/data
sudo chgrp www-data /var/www/data
sudo chmod 754 /var/www/data
ls -ld /var/www/data
```

Ausgabe:
```bash
drwxr-xr--
```

- user (u): lesen, schreiben, ausführen → 7
- group (g): lesen, ausführen → 5
- others (o): nur lesen, Inhalt auflisten → 4

others kann mit `ls` den Inhalt auflisten, aber keine Dateiinhalte
lesen da kein `r` auf die Dateien selbst gesetzt ist.
group `www-data` kann Dateiinhalte lesen aber nicht verändern da
kein `w` gesetzt ist.

---

#### Erweiterung 1 – Sticky Bit
User der Gruppe www-data sollen Dateien anderer User nicht löschen
oder umbenennen können.

```bash
sudo chmod +t /var/www/data
ls -ld /var/www/data
```

Ausgabe:
```bash
drwxr-xr-T
```

Das Sticky Bit (T) verhindert dass User Dateien anderer User im
Verzeichnis löschen oder umbenennen können, auch wenn sie
Schreibrechte auf das Verzeichnis haben.
Das große `T` bedeutet dass others kein `x` Recht hat, bei einem
kleinen `t` wäre `x` für others gesetzt.

---

#### Erweiterung 2 – SetGID Bit
Alle neuen Dateien/Verzeichnisse in data sollen automatisch der
Gruppe www-data zugeordnet werden.

```bash
sudo chmod g+s /var/www/data
ls -ld /var/www/data
```

Ausgabe:
```bash
drwxr-sr-T
```

Das SetGID Bit (s) sorgt dafür dass alle neuen Dateien und
Verzeichnisse die in /var/www/data erstellt werden automatisch
die Gruppe www-data erben.
