# Linux Permissions Übungen 2

## 6 Übung (Webserver Directory Permissions II)

### Aufgabenstellung

In dieser Aufgabe soll unter Kali Linux im Verzeichnis `/var/www` ein neues Verzeichnis `data` erstellt werden. Dieses Verzeichnis soll so konfiguriert werden, dass nur Benutzer der Gruppe `www-data` Zugriff auf die Inhalte haben.

Zusätzlich soll gelten:
- Andere Benutzer dürfen den Verzeichnisinhalt sehen, aber keine Dateien lesen
- Benutzer der Gruppe `www-data` dürfen Dateien lesen, aber nicht verändern
- Das Löschen oder Umbenennen von Dateien durch Gruppenmitglieder soll verhindert werden
- Alle neu erstellten Dateien im Verzeichnis sollen automatisch der Gruppe `www-data` zugewiesen werden

Die Umsetzung soll getestet werden.

---

### Lösung

#### 1. Verzeichnis erstellen
Zuerst wird das Verzeichnis unter `/var/www` angelegt:

```bash
sudo mkdir -p /var/www/data
```

#### 2. Besitzer und Gruppe setzen

Das Verzeichnis wird dem Benutzer `root` und der Gruppe `www-data` zugewiesen:

```bash
sudo chown root:www-data /var/www/data
```

#### 3. Rechte für das Verzeichnis setzen

Damit die Anforderungen erfüllt werden, werden spezielle Rechte gesetzt:

```bash
sudo chmod 3775 /var/www/data
```

Erklärung:

* `3` --> setgid + sticky bit aktiviert
* `7` --> Besitzer (root) hat volle Rechte
* `7` --> Gruppe (www-data) hat volle Rechte im Verzeichnis (Betreten + Auflisten)
* `5` --> andere Benutzer können das Verzeichnis betreten und Inhalte auflisten, aber keine Dateien lesen (Dateirechte bestimmen den Zugriff auf Inhalte)


#### 4. Testdatei erstellen

Zum Testen wird eine Datei im Verzeichnis angelegt:

```bash
sudo touch /var/www/data/test.txt
echo "Hallo Test" | sudo tee /var/www/data/test.txt
```

Danach werden Besitzer und Rechte gesetzt:

```bash
sudo chown root:www-data /var/www/data/test.txt
sudo chmod 640 /var/www/data/test.txt
```


### Test

#### Benutzer ohne Gruppenmitgliedschaft (user1)

```bash
su user1
ls /var/www/data
```

--> Der Dateiname wird angezeigt

```bash
cat /var/www/data/test.txt
```

-->  Zugriff wird verweigert (Permission denied)

---

#### Benutzer mit Gruppenmitgliedschaft (user2)

```bash
su user2
ls /var/www/data
cat /var/www/data/test.txt
```

--> Dateiliste wird angezeigt
--> Inhalt der Datei wird ausgegeben
